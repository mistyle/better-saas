# 权限管理快速参考

## 核心概念

这是一个简化的权限管理系统，专注于**管理员权限检查**，支持**双重管理员判断机制**。

## 双重管理员判断

### 方式一：数据库 role 字段
```sql
-- 用户表结构
CREATE TABLE user (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL UNIQUE,
  role TEXT,  -- 'admin' 表示管理员
  -- ... 其他字段
);
```

### 方式二：环境变量
```env
# .env.local
ADMIN_EMAILS=admin@example.com,manager@example.com
```

### 判断逻辑
系统会检查以下两个条件，**任一满足即为管理员**：
1. 数据库中 `user.role === 'admin'`
2. 用户邮箱在 `ADMIN_EMAILS` 环境变量中

## 管理员设置

### 推荐方式：使用脚本
```bash
# 设置用户为管理员（会更新数据库 role 字段）
pnpm tsx scripts/setup-admin.ts admin@example.com
```

### 备用方式：环境变量
```env
# 适用于开发阶段或应急情况
ADMIN_EMAILS=admin@example.com,emergency@example.com
```

### 直接数据库操作
```sql
-- 仅在必要时使用
UPDATE user SET role = 'admin' WHERE email = 'admin@example.com';
```

## 基本使用

### 1. 在组件中检查管理员权限

```tsx
import { useIsAdmin } from '@/components/auth/permission-provider'

export function AdminPanel() {
  const isAdmin = useIsAdmin()
  
  if (!isAdmin) {
    return <div>需要管理员权限</div>
  }
  
  return <div>管理员内容</div>
}
```

### 2. 条件渲染

```tsx
export function Navigation() {
  const isAdmin = useIsAdmin()
  
  return (
    <nav>
      <Link href="/settings">设置</Link>
      {isAdmin && <Link href="/admin">管理员</Link>}
    </nav>
  )
}
```

### 3. 使用权限守卫

```tsx
import { AdminGuard } from '@/components/admin-guard'

export default function AdminPage() {
  return (
    <AdminGuard>
      <h1>管理员专用页面</h1>
    </AdminGuard>
  )
}
```

## 可用的 Hooks

### `useIsAdmin()`
- **返回**：`boolean`
- **用途**：检查当前用户是否为管理员

```tsx
const isAdmin = useIsAdmin()
```

### `useUserRole()`
- **返回**：`'admin' | 'user'`
- **用途**：获取用户角色（为了向后兼容）

```tsx
const role = useUserRole()
```

### `useHasPermission()`
- **返回**：`(permission: string) => boolean`
- **用途**：检查用户是否有特定权限

```tsx
const hasPermission = useHasPermission()
const canViewSettings = hasPermission('settings.view')
```

## 权限逻辑

### 管理员权限
- **数据库方式**：`user.role === 'admin'`
- **环境变量方式**：邮箱在 `ADMIN_EMAILS` 中
- **权限范围**：拥有所有权限

### 普通用户权限
- `settings.view` - 查看设置
- `profile.edit` - 编辑个人资料  
- `billing.view` - 查看账单

## 常见模式

### 早期返回模式
```tsx
export function AdminComponent() {
  const isAdmin = useIsAdmin()
  
  if (!isAdmin) {
    return <AccessDenied />
  }
  
  return <AdminContent />
}
```

### 条件渲染模式
```tsx
export function Dashboard() {
  const isAdmin = useIsAdmin()
  
  return (
    <div>
      <h1>仪表板</h1>
      {isAdmin && <AdminTools />}
    </div>
  )
}
```

### 权限检查模式
```tsx
export function UserProfile() {
  const hasPermission = useHasPermission()
  
  return (
    <div>
      <h1>用户资料</h1>
      {hasPermission('profile.edit') && <EditButton />}
    </div>
  )
}
```

## 组件架构

```
PermissionWrapper (Server Component)
├─ 获取管理员状态（双重检查）
│  ├─ 检查数据库 role 字段
│  └─ 检查环境变量 ADMIN_EMAILS
└─ PermissionProvider (Client Component)
   ├─ 提供 React Context
   └─ 子组件使用 Hooks
```

## 注意事项

1. **双重验证机制**
   - 任一方式通过即为管理员
   - 数据库方式优先级更高（推荐生产使用）
   - 环境变量作为备用方案

2. **必须在 PermissionProvider 内部使用**
   - 所有权限 Hooks 必须在 `PermissionProvider` 包裹的组件中使用

3. **服务器端安全**
   - 权限检查在服务器端进行
   - 客户端无法访问敏感环境变量

4. **性能优化**
   - 零额外 API 请求
   - 权限信息预先计算

## 故障排除

### Hook 使用错误
```
Error: useIsAdmin must be used within a PermissionProvider
```
**解决方案**：确保组件在 `PermissionProvider` 内部使用

### 权限检查失败
**检查列表**：
- [ ] 数据库中用户的 `role` 字段是否为 `'admin'`
- [ ] 环境变量 `ADMIN_EMAILS` 是否正确配置
- [ ] 用户邮箱是否在管理员列表中
- [ ] 用户是否已正确登录

### 管理员设置失败
**使用脚本设置时的检查**：
- [ ] 邮箱是否在 `ADMIN_EMAILS` 环境变量中
- [ ] 用户是否已注册并存在于数据库中
- [ ] 数据库连接是否正常

### 环境变量问题
- 确保 `.env.local` 文件存在
- 重启开发服务器以加载新的环境变量
- 检查邮箱格式是否正确（用逗号分隔）

## 管理员管理工作流

### 新用户成为管理员
1. 用户先注册账号
2. 将邮箱添加到 `ADMIN_EMAILS` 环境变量
3. 运行设置脚本：`pnpm tsx scripts/setup-admin.ts user@example.com`
4. 用户重新登录即可获得管理员权限

### 移除管理员权限
```sql
-- 移除数据库中的管理员角色
UPDATE user SET role = NULL WHERE email = 'user@example.com';
```
```env
# 从环境变量中移除邮箱
ADMIN_EMAILS=admin@example.com  # 移除 user@example.com
```

### 批量设置管理员
```bash
# 可以编写批量脚本
emails=("admin1@example.com" "admin2@example.com" "admin3@example.com")
for email in "${emails[@]}"; do
  pnpm tsx scripts/setup-admin.ts "$email"
done
```
