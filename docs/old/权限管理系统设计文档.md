# 权限管理系统设计文档

## 概述

本文档描述了 Next.js 应用中的权限管理系统设计。系统专注于**简单高效**的管理员权限检查，支持**双重管理员判断机制**。

## 设计原则

### 核心理念
- **简单优先**：只处理管理员权限检查，不进行复杂的权限计算
- **双重验证**：支持数据库 role 字段和环境变量两种管理员判断方式
- **服务器计算，客户端消费**：权限判断在服务器端完成，客户端通过 Context 消费
- **零额外请求**：通过 React Server Components 预先获取权限信息

### 技术选择
- **React Server Components**：利用服务器端渲染的优势
- **React Context**：在客户端组件间共享权限状态
- **Better Auth**：用户认证和会话管理
- **Drizzle ORM**：数据库操作和类型安全

### 组件职责

#### 1. Server Actions (`src/server/actions/auth-actions.ts`)
```typescript
export async function getUserAdminStatus(): Promise<boolean>
```
- **职责**：在服务器端安全地检查用户是否为管理员
- **优势**：可以访问服务器端环境变量和数据库，安全性高
- **返回**：布尔值，表示用户是否为管理员

#### 2. PermissionWrapper (`src/components/auth/permission-wrapper.tsx`)
```tsx
export default async function PermissionWrapper({ children })
```
- **职责**：服务器组件，获取管理员状态并传递给客户端
- **特点**：在服务器端执行，无需额外的 API 调用

#### 3. PermissionProvider (`src/components/auth/permission-provider.tsx`)
```tsx
export function PermissionProvider({ children, isAdmin })
```
- **职责**：客户端 Context Provider，管理权限状态
- **特点**：轻量级，只管理管理员状态

#### 4. 权限 Hooks
```tsx
export function useIsAdmin(): boolean
export function useUserRole(): string
export function useHasPermission(): (permission: string) => boolean
```
- **职责**：为客户端组件提供便捷的权限检查方法
- **特点**：简单易用，基于管理员状态提供权限逻辑

## 实现细节

### 1. 双重管理员判断机制

#### 方式一：数据库 role 字段
```typescript
// 数据库 schema
export const user = pgTable('user', {
  // ... 其他字段
  role: text('role'), // 'admin' 或 null
  // ... 其他字段
});

// 权限检查
if (user.role === 'admin') {
  return true;
}
```

#### 方式二：环境变量配置
```env
# .env.local
ADMIN_EMAILS=admin@example.com,manager@example.com
```

```typescript
// 权限检查
if (user.email) {
  const adminEmails = getAdminEmails();
  return adminEmails.includes(user.email);
}
```

#### 综合判断逻辑
```typescript
export function isAdmin(user: User | null): boolean {
  if (!user) {
    return false;
  }
  
  // 方式1: 检查数据库中的 role 字段
  if (user.role === 'admin') {
    return true;
  }
  
  // 方式2: 检查邮箱是否在环境变量中
  if (user.email) {
    const adminEmails = getAdminEmails();
    return adminEmails.includes(user.email);
  }
  
  return false;
}
```

### 2. 管理员设置脚本

系统提供了便捷的管理员设置脚本：

```bash
# 设置用户为管理员
pnpm tsx scripts/setup-admin.ts admin@example.com
```

脚本功能：
- 检查邮箱是否在 `ADMIN_EMAILS` 环境变量中
- 查找数据库中的用户记录
- 将用户的 `role` 字段设置为 `'admin'`

### 3. 类型扩展

为了支持数据库字段，我们扩展了 Better Auth 的用户类型：

```typescript
// src/types/index.d.ts

```

### 4. 使用示例

#### 在布局中集成
```tsx

export default function ProtectedLayout({ children }) {
  return (
    <PermissionWrapper>
      <div className="dashboard-layout">

      </div>

```

#### 在组件中使用
```tsx

```

## 管理员管理流程

### 1. 通过环境变量设置（临时）
```env
ADMIN_EMAILS=admin@example.com,manager@example.com
```
- **优点**：快速设置，无需数据库操作
- **缺点**：需要重启应用，不够灵活

### 2. 通过数据库设置（推荐）
```bash
# 使用脚本设置
pnpm tsx scripts/setup-admin.ts admin@example.com

# 或直接更新数据库
UPDATE user SET role = 'admin' WHERE email = 'admin@example.com';
```
- **优点**：灵活，无需重启应用
- **缺点**：需要数据库访问权限

### 3. 混合使用（最佳实践）
1. **开发阶段**：使用环境变量快速设置
2. **生产环境**：使用数据库 role 字段管理
3. **应急情况**：环境变量作为备用方案

## 性能优化

### 1. 零额外请求
- 权限信息在服务器端预先获取
- 通过 React Server Components 传递给客户端
- 避免了客户端的额外 API 调用

### 2. 简化的权限逻辑
- 只进行管理员状态检查，避免复杂的权限计算
- 双重检查机制提供了灵活性而不牺牲性能
- 减少了服务器端的计算负担

### 3. Context 优化
- 使用 React Context 在客户端组件间共享状态
- 避免了重复的权限检查
- 减少了组件重渲染

## 安全考虑

### 1. 服务器端验证
- 所有权限检查都在服务器端进行
- 客户端无法直接访问环境变量
- 防止了权限信息的篡改

### 2. 双重验证机制
- 数据库 role 字段提供持久化的权限管理
- 环境变量提供应急和开发阶段的灵活性
- 任一方式通过即可获得管理员权限

### 3. 会话管理
- 依赖 Better Auth 的会话管理
- 自动处理会话过期和刷新
- 确保权限检查的准确性

### 4. 错误处理
- 权限检查失败时返回安全的默认值
- 避免了权限泄露的风险
- 提供了清晰的错误信息

## 扩展性

### 1. 添加更多角色
如果将来需要更多角色类型，可以：
- 扩展数据库 role 字段的值（如 'admin', 'moderator', 'user'）
- 更新 `isAdmin` 函数支持不同角色检查
- 创建新的 Hook 来处理角色权限

### 2. 基于组织的权限
- 可以扩展为基于组织的权限系统
- 在权限检查中添加组织上下文
- 支持不同组织的管理员权限

### 3. 细粒度权限控制
- 可以添加权限表来支持更细粒度的权限控制
- 保持当前的管理员检查作为基础权限
- 在管理员权限基础上添加特定功能权限

## 故障排除

### 1. 权限检查失败
- 检查数据库中用户的 `role` 字段是否为 `'admin'`
- 检查环境变量 `ADMIN_EMAILS` 是否正确配置
- 确认用户邮箱是否在管理员列表中
- 验证用户是否已正确登录

### 2. 管理员设置失败
- 确保邮箱在 `ADMIN_EMAILS` 环境变量中
- 检查用户是否已注册并存在于数据库中
- 验证数据库连接是否正常

### 3. Hook 使用错误
- 确保在 `PermissionProvider` 内部使用权限 Hook
- 检查组件是否在正确的布局中

### 4. 类型错误
- 确保 `src/types/index.d.ts` 中的类型扩展正确
- 重启 TypeScript 服务器以加载新的类型定义

## 总结

这个权限管理系统通过**双重管理员判断机制**，既保持了简单性，又提供了灵活性：

### 🎯 核心优势
- **简单高效**：专注于管理员权限检查，避免过度复杂
- **双重保障**：数据库 + 环境变量两种验证方式
- **安全可靠**：服务器端验证，防止权限篡改
- **易于管理**：提供脚本工具，支持灵活的权限设置

### 🚀 适用场景
- **开发阶段**：使用环境变量快速设置测试管理员
- **生产环境**：使用数据库 role 字段进行持久化管理
- **应急情况**：环境变量作为备用管理员设置方案

这种设计既满足了当前的需求，又为未来的扩展留出了空间，是一个实用且高效的解决方案。
