# 测试文档

本项目包含完整的测试套件，涵盖单元测试、集成测试、端到端测试、性能测试和安全测试。

## 🧪 测试类型

### 1. 单元测试 (Unit Tests)

使用 Jest + React Testing Library 进行单元测试。

**覆盖范围:**
- 工具函数 (`src/lib/utils.ts`)
- 权限系统 (`src/lib/auth/permissions.ts`)
- 文件服务 (`src/lib/file-service.ts`)
- 自定义 Hooks (`src/hooks/`)
- UI 组件 (`src/components/ui/`)
- 状态管理 (`src/store/`)

**运行命令:**
```bash
pnpm test:unit          # 运行单元测试
pnpm test:watch         # 监视模式
pnpm test:coverage      # 生成覆盖率报告
```

### 2. 集成测试 (Integration Tests)

测试不同模块之间的交互。

**覆盖范围:**
- API 路由测试
- 数据库操作测试
- 第三方服务集成测试
- 文件上传服务测试

**运行命令:**
```bash
pnpm test:integration   # 运行集成测试
```

### 3. 端到端测试 (E2E Tests)

使用 Playwright 进行端到端测试。

**覆盖范围:**
- 用户认证流程
- 仪表板导航和功能
- 文件管理操作
- 管理员功能
- 支付流程
- 多语言支持
- 响应式设计

**运行命令:**
```bash
pnpm test:e2e           # 运行 E2E 测试
pnpm test:e2e:ui        # 使用 UI 模式
pnpm test:e2e:headed    # 有头模式（显示浏览器）
```

### 4. 性能测试 (Performance Tests)

**包含内容:**
- Lighthouse 性能审计
- 负载测试 (Artillery)
- 性能基准测试
- Core Web Vitals 测试

**运行命令:**
```bash
pnpm test:performance   # 运行性能基准测试
pnpm test:lighthouse    # 运行 Lighthouse 审计
pnpm test:load          # 运行负载测试
```

### 5. 安全测试 (Security Tests)

**覆盖范围:**
- 认证和授权测试
- 输入验证测试
- CSRF 防护测试
- XSS 防护测试
- SQL 注入防护测试
- 速率限制测试
- HTTP 安全头测试

**运行命令:**
```bash
pnpm test:security      # 运行安全测试
```

## 📝 编写测试指南

### 单元测试示例

```typescript
import { describe, it, expect } from '@jest/globals'
import { render, screen } from '@testing-library/react'
import { Button } from '@/components/ui/button'

describe('Button组件', () => {
  it('应该渲染按钮文本', () => {
    render(<Button>Click me</Button>)
    expect(screen.getByRole('button', { name: 'Click me' })).toBeInTheDocument()
  })
})
```

### E2E 测试示例

```typescript
import { test, expect } from '@playwright/test'

test('用户登录流程', async ({ page }) => {
  await page.goto('/auth/sign-in')
  await page.fill('input[type="email"]', 'test@example.com')
  await page.fill('input[type="password"]', 'password123')
  await page.click('button[type="submit"]')
  await expect(page).toHaveURL('/dashboard')
})
```

## 📈 持续集成

测试套件已配置为在 CI/CD 流水线中运行：

1. **代码提交时**: 运行单元测试和集成测试
2. **Pull Request**: 运行完整测试套件
3. **部署前**: 运行性能和安全测试
4. **定期**: 运行负载测试和安全扫描

## 🔍 测试最佳实践

1. **测试命名**: 使用描述性的测试名称
2. **测试隔离**: 每个测试应该独立运行
3. **数据清理**: 测试后清理测试数据
4. **模拟外部依赖**: 使用 MSW 模拟 API 调用
5. **可读性**: 测试代码应该易于理解
6. **覆盖率**: 关注重要的业务逻辑
7. **性能**: 保持测试运行速度
8. **维护**: 定期更新和重构测试
