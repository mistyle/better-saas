### 1. TypeScript 类型安全错误

#### 问题
- `currentAccount` 可能为 `undefined` 的类型错误
- Stripe webhook 中 `invoice.id` 可能为 `undefined`

#### 解决方案
```typescript
// 修复前
const currentAccount = account[0];
const newBalance = currentAccount.balance + amount;

// 修复后
const currentAccount = account[0];
if (!currentAccount) {
  throw new Error('Failed to retrieve credit account after creation');
}
const newBalance = currentAccount.balance + amount;
```

```typescript
// 修复前
invoice.id

// 修复后
invoice.id || 'unknown'
```

### 2. React Hooks 依赖问题

#### 问题
- `useEffect` 缺少依赖项
- 函数未使用 `useCallback` 包装导致依赖循环

#### 解决方案
```typescript
// 修复前
const fetchTransactions = async () => { /* ... */ };
useEffect(() => {
  fetchTransactions();
}, [limit]);

// 修复后
const fetchTransactions = useCallback(async () => { /* ... */ }, [limit]);
useEffect(() => {
  fetchTransactions();
}, [fetchTransactions]);
```

### 3. 代码风格问题

#### 问题
- 使用数组索引作为 React key
- 不必要的 else 子句
- 使用 forEach 而非 for...of
- 不必要的模板字符串

#### 解决方案
```typescript
// 修复前
{Array.from({ length: 4 }).map((_, i) => (
  <Card key={i}>

// 修复后
{Array.from({ length: 4 }).map((_, i) => (
  <Card key={`balance-card-${i}`}>
```

```typescript
// 修复前
if (condition) {
  return value;
} else {
  return otherValue;
}

// 修复后
if (condition) {
  return value;
}
return otherValue;
```

```typescript
// 修复前
errors.forEach(error => {
  console.log(error);
});

// 修复后
for (const error of errors) {
  console.log(error);
}
```

### 4. 数据库查询类型错误

#### 问题
- Drizzle ORM 的 `sql` 模板使用方式不正确

#### 解决方案
```typescript
// 修复前
const result = await db.execute(sql`SELECT COUNT(*) as count FROM "user"`);

// 修复后
const query = sql`SELECT COUNT(*) as count FROM "user"`;
const result = await db.execute(query);
```
